MercadoPago iOS SDK

Fácil, rápido, simple. La mejor experiencia de pagos en tu aplicación.

Integrá el checkout de MercadoPago en pocos minutos.

Te damos todo resuelto, solo indicanos qué querés cobrar y nos encargaremos del resto. A continuación podés ver un video de cinco minutos donde seguirás paso a paso la documentación y como resultado tendrás integrado MercadoPago en una aplicación de ejemplo.


1. Agregá la dependencia a tu proyecto

Si en tu proyecto usas CocoaPods, podés agregar la dependencia en el Podfile del módulo donde nos integres.

Si no utilizas CocoaPods, descargá la SDK de MercadoPago en este link y agregala a tu proyecto.

                     
                        source 'https://github.com/CocoaPods/Specs.git'
                        # Se necesita este parámetro por ser una SDK en swift
                        use_frameworks!
                        platform :ios, '8.0'
                        pod 'MercadoPagoSDK', '~> 0.1.0'

2. Setea tu clave pública

Setea la clave pública correspondiente a tu cuenta de MercadoPago en la clase AppDelegate. En este caso utilizaremos una clave de prueba. Más adelante veremos como puedes obtener tu propia clave pública.

Si tu proyecto está escrito en Objective-C necesitas setear en "Build Settings" el parámetro "Embedded Content Contains Swift Code" en "Yes", ya que la SDK está escrita en Swift. 

                     
                      class AppDelegate: UIResponder, UIApplicationDelegate {

                      var window: UIWindow?
                      // 1 - Clave pública
                      let publicKey = "TEST-ad365c37-8012-4014-84f5-6c895b3f8e0a"

                      func application(application: UIApplication, 
                          didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool { 
                            MercadoPagoContext.setPublicKey(publicKey)
                            return true
                      }

3. Agregá un botón de pago en tu aplicación

A modo de ejemplo proponemos que inicies el flujo de MercadoPago desde un botón.

Creá un ViewController (MainViewController, por ejemplo) e insertá un botón en el .xib correspondiente. También agregá un campo de texto (en nuestro caso lo llamamos paymentResult) para mostrar los resultados del pago.

Pegá el código de la derecha en tu clase MainViewController.swift.

En el siguiente paso estarás trabajando sobre el evento asociado al click botón (startCheckout). 

                     
                      import UIKit

                      class MainViewController: UIViewController {

                          @IBOutlet weak var payButton: UIButton!

                          @IBOutlet weak var paymentResult: UITextField!
    
                          override func viewDidLoad() {
                              super.viewDidLoad()
                              self.payButton.addTarget(self, 
                                action: #selector(MainViewController.startCheckout), 
                                forControlEvents: .TouchUpInside)
                          }
                      }

4. Iniciá el Checkout de MercadoPago desde tu aplicación

* Atención: En el código de la derecha te ofrecemos datos de prueba para que puedas probar la experiencia completa, pero recuerda que cuando estés en producción (Paso 6) vas a tener que generarlos. Es simple, no te desanimes.

Para iniciar nuestro checkout sólo necesitas:

  • Clave pública: Es un identificador que agrupa tu cuenta, tu aplicación y sus configuraciones. En este link podés conocer tus credenciales y aquí podés ver cómo crear y configurar aplicaciones.
  • Identificador de la preferencia de pago: Es una representación de lo que estás cobrando. Incluye, entre otros, los siguientes atributos:
    • Datos y monto del producto que estás vendiendo.
    • Medios de pago que aceptas.
    • Información del comprador (el email es requerido por prevención de fraude y protección durante y después del pago)
    •                      
                      // 2 - Identificador de preferencia de pago
                      let prefId = "150216849-ceed1ee4-8ab9-4449-869f-f4a8565d386f" 

                      // Método ejecutado al hacer clic en el botón
                      func startCheckout(){
                        let checkout = MPFlowBuilder.startCheckoutViewController(prefId) 
                        { (payment : Payment) in
                          // Listo! El pago ya fue procesado por MP. 
                          // En el paso siguiente se describe el método displayPaymentInfo
                          self.displayPaymentInfo(payment)
                        }
        
                        self.presentViewController(checkout, animated: true, completion: {})
                      }

    * A modo de prueba, estamos usando una preferencia cuyo producto es un Bajo de $1000 y acepta todos los medios de pago disponibles en MercadoPago.

    5. Esperá el resultado del pago

    Una vez finalizado el flujo, el SDK retornará el pago que realizó tu cliente en la acción de callback seteada como segundo parámetro en el método de MPFlowBuilder.startCheckoutViewController.

    Si algún error irrecuperable sucediera dentro del flujo o tu usuario lo abandonara, el SDK retornará un objeto de error para que puedas entender qué sucedió.

    Los atributos más importante del pago son:

    • id: Identificador del pago.
    • status: Estado del pago (Aquí podrás ver los posibles estados).
    • payment_method_id: Identificador del medio de pago seleccionado por tu usuario.
    • payment_type_id: El tipo de medio seleccionado.
    • card: Objeto que identifica la tarjeta de tu usuario.
    • issuer_id: Identificador del banco (si tu usuario usó una tarjeta).
    • installments: Número de cuotas seleccionadas (si tu usuario usó una tarjeta).

    Conocé más sobre el objeto Payment aquí. 

                         
                      func displayPaymentInfo(payment : Payment){
                          let paymentInfo : String
                          paymentInfo = "id : " + String(payment._id) + ", 
                              status : " + payment.status + ", 
                              status detail : " + payment.statusDetail
                          self.paymentResult.text = paymentInfo
                      }

    6. Testeá nuestro checkout

    Como te contamos en el paso 4, hasta aquí utilizamos datos de prueba.

    A la derecha podes ver un listado de tarjetas para probar nuestro checkout y prefijos para el nombre del portador de la tarjeta de modo que puedas obtener pagos con distintos resultados.

    Ofrecemos una herramienta para crear preferencias con distintas configuraciones para que puedas trabajar en un entorno de prueba. Las configuraciones que permiten las prefrencias son:

    • Excluír medios de pago.
    • Excluír tipos de pago (sólo tarjeta, por ejemplo).
    • Indicar un medio de pago por defecto (la tarjeta que más usa tu cliente).
    • Limitar la cantidad de cuotas.
    • Seleccionar las cuotas a pagar por defecto.

    7. Poné tu aplicación en producción

    Antes de salir a producción asegurate:

    • Tener o crear una cuenta de MercadoPago dónde recibirás el dinero.
    • Crear una aplicación.
    • Configurar tus credenciales.
    • Crear una Preferencia de pago desde tu servidor.

    ¿Qué tenes que hacer desde tu servidor?
    Desde tu servidor, sólo tienes que crear la preferencia de pago y retornar la misma respuesta que te dan nuestros servicios. La preferencia debe ser creada desde tu servidor ya que debes firmarla con tu clave privada. De esta forma nos aseguramos estar protegiendo tanto al comprador, como a tu propio usuario vendedor. Aquí te contamos cómo crear preferencias desde tu servidor.

    Creá una preferencia de pago tan completa como puedas. Mientras más información nos envíes, mejor será tu experiencia y la de tus usuarios.

    ¿Y desde tu aplicación?
    Te ofrecemos una clase llamada MerchantServer con métodos que se conectan con tu servidor. En este caso, el método createPreference hace un POST y envía como cuerpo del mensaje el diccionario que hayas definido (preferenceBody). Por eso debes indicarnos tu URL base (https://api.tunombre.com) y la uri (/create_preference) donde esperas los datos para crear la preferencia.
    MerchantServer se encargará de transformar la respuesta de tu servicio (la misma que los servicios de MercadoPago) en un objeto CheckoutPreference cuyo ID es el punto de entrada a nuestro checkout. 

                         
                        let preferenceBody = ["amount" : 12, "itemId" : 29334, "customerId": 207, "payer_email" : "customermail@test.com"]
        
                        // Seteo de URL y preferenceUri
                        MercadoPagoContext.setBaseURL("https://api.tunombre.com")
                        MercadoPagoContext.setPreferenceURI("/create_preference")

                              // La preferencia se creó correctamente.

                              })
                          }, failure: { (error) in
                              // Ups, ocurrió un error
                        })
